梳理好了博客的发布链条
2022-03-10

博客的发布链条

  1. 本地通过 Hexo 来管理一系列的 Markdown 文档。
  2. 每一次更新博文后，通过 Hexo 来动态生成网站。
  3. 使用 hexo-deployer-git 插件来把生成结果推送到 Github.
  4. 在 Plesk 虚拟主机上设置 Git 拉取，并在 Github 仓库设置 Plesk 提供的 Webhook 链接，使虚拟主机在每次 Github 博文仓库发生推送后及时拉取。

尾巴

这样的话，我每次写完后，直接运行 hexo clean && hexo deploy 就会发布到 Github ，然后 Plesk 虚拟主机就会拉取最新博文，支撑网站的访问。考虑到稳定性，没有使用 Github Pages 服务，而是自己买虚拟主机，并且这次不再使用虚拟主机作为写作平台，而只充当发布平台。
